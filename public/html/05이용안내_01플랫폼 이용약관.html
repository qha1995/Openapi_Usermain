제1장 총칙

제1조(목적)

이 약관은「전자정부법」 제21조, 같은 법 시행령 제17조 및 「민간서비스 활용을 통한 전자적 대민서비스 제공 등에 관한 규정」(이하 “규정”이라 합니다.)에 따라 디지털서비스개방 이용에 관한 사항을 규정함을 목적으로 합니다.

제2조(정의)

이 약관에서 사용하는 용어의 뜻은 다음과 같습니다

  1. 1. “디지털서비스개방”(이하 “플랫폼”이라 합니다)이란 민간서비스 활용을 지원하기 위하여 행정안전부장관이 구축·관리하는 정보시스템을 말합니다.
  2. 2. “전자적 대민서비스”(이하 “디지털서비스”라 합니다)란 중앙행정기관등이 「전자정부법」(이하 “법”이라 한다) 제2조제5호에 따른 전자정부서비스 중 국민, 기업 등에 제공하는 서비스를 말합니다.
  3. 3. “민간디지털서비스”(이하 “민간서비스”라 한다)란 법 제21조제1항에 따른 개인 및 기업, 단체 등의 서비스 중 웹 또는 모바일 앱을 통해 국민, 기업 등에 제공하는 서비스를 말합니다.
  4. 4. “수요기관”이란 개방된 디지털서비스를 활용하여 이용자에게 각종 조회ㆍ신청ㆍ등록 및 제공 등의 서비스를 제공하는 기관을 말합니다.
  5. 5. “개방기관”이란 소관 디지털서비스를 개방하는 기관을 말합니다.
  6. 6. “운영기관”이란 플랫폼의 구축·운영을 지원하는 기관으로서 이 약관에서는 「지능정보화기본법」에 따른 한국지능정보사회진흥원을 말합니다.
  7. 7. “이용자”란 디지털서비스를 최종적으로 이용하는 국민, 기업 등을 말합니다.
  8. 8. “API”란 Application Programming Interface의 약자로서 개방기관과 수요기관간 다양한 디지털서비스의 조회ㆍ신청ㆍ등록 및 제공 등에 필요한 정보를 송수신하는 프로그램을 말합니다.
  9. 9. “API key”란 API 서비스를 이용할 때 적법한 권한을 가진 기관의 호출여부 등을 검증하는 식별자를 말합니다.
  10. 10. “API 서비스”란 개방기관이 민간서비스를 통하여 소관 전자적 대민서비스를 제공할 수 있도록 API 방식으로 플랫폼에 등록제공하는 서비스를 말합니다.
  11. 11. “디지털서비스 개방”(이하 “포털”이라 합니다)이란 플랫폼이 제공하는 웹서비스를 말합니다.
  12. 12. 그 밖에 이 약관에서 정의하지 않은 용어는 「전자정부법」, 「전자정부법 시행령」 및 「민간서비스 활용을 통한 전자적 대민서비스 제공 등에 관한 규정」에서 정한 정의나 일반적인 개념에 따릅니다.

제3조(약관의 효력 및 개정)

  1. ① 이 약관은 수요기관이 디지털서비스개방에서 제공하는 서비스를 이용하기 위하여 본 약관에 동의하고 운영기관에 공문형태로 제출 함으로써 효력이 발생합니다.
  2. ② 행정안전부장관은 「약관의 규제에 관한 법률」 등 관련 법령에 위배되지 않는 범위 내에서 이 약관을 개정할 수 있으며, 개정 약관 적용일의 1개월 전부터 약관의 개정 내용을 수요기관에게 사전 공지하여야 합니다. 다만, 긴급하게 약관을 개정할 필요가 있는 경우 1개월보다 짧은 기간 전에 공지할 수 있으나, 개정전 공지기간은 7일을 초과할 수 있도록 하여야 합니다.
  3. ③ 제2항에 따른 약관 개정 공지 후 수요기관 또는 개방기관의 장이 개정 약관 적용일까지 명시적으로 약관 개정에 대한 거부의사를 표시하지 아니하면 개정 약관에 동의한 것으로 봅니다.
  4. ④ 이용기관 또는 개방기관은 약관 개정이 필요한 경우에 행정안전부장관에게 약관 개정을 요청할 수 있으며, 행정안전부장관은 개정 요청 부분을 검토하여 제2항에 따라 약관을 개정할 수 있습니다.

제2장 디지털서비스개방의 이용

제4조(디지털서비스개방의 서비스 제공 범위)

  1. ① 플랫폼의 서비스 제공 범위는 다음 각 호와 같습니다.
    1. 1. 플랫폼 이용을 위한 기관회원의 등록ㆍ삭제 및 회원정보 관리
    2. 2. API 서비스를 위한 API 목록 및 기능 정보 제공
    3. 3. 플랫폼 접속을 위한 API key의 발급ㆍ관리
    4. 4. 현황 모니터링 및 통계 조회
    5. 5. 그 밖에 민간서비스 활용 지원을 위한 기능
  2. ② 제1항에 따른 서비스 제공범위에 변동이 있는 경우에는 해당 내용을 포털에 게시하고, 수요기관 및 개방기관의 업무담당자에게 사전에 통지하여야 합니다.

제5조(플랫폼 이용신청 및 사용자 등록)

  1. ① 수요기관이 플랫폼을 이용하기 위해서는 플랫폼을 사용할 담당자와 수요기관의 기본정보를 별지 제1호에 따른 플랫폼 이용신청서에 기재하여 별지 제2호에 따른 개인정보 수집·이용동의서와 함께 운영기관에 제출하는 방식으로 플랫폼 이용신청을 하여야 합니다.
  2. ② 제1항에 따른 이용신청을 받은 운영기관은 신청받은 내용을 승인하고 제출받은 기본정보 내용을 플랫폼에 등록하여야 합니다.
  3. ③ 제2항에 따른 등록 후 담당자는 최초로 플랫폼에 접속하여 연락처와 전자우편주소 등 추가정보를 입력하고 본인 인증을 하여야 합니다.
  4. ④ 수요기관은 업무담당자 중 1인을 대표관리자로 지정하여, 플랫폼 포털에 등록된 담당자의 현행화 관리에 대한 책임을 집니다.
  5. ⑤ 플랫폼 이용신청 절차에 관한 상세 내용은 포털에 게시된 이용안내 등의 내용에 따릅니다.

제6조(접속토큰과 이용토큰 발급)

  1. ① 수요기관은 플랫폼 이용신청이 승인된 이후 포털에 수요기관 서버 정보 및 IP 정보 등을 입력하고 접속토큰(임시 KEY) 발급을 신청합니다.
  2. ② 수요기관은 발급받은 임시 KEY를 사용하여 플랫폼 수요기관 인증절차를 수행한 후 이용토큰(운영 KEY)을 발급받을 수 있습니다.
  3. ③ 아래 각 호의 경우에는 기존에 발급받은 이용토큰(운영 KEY)를 폐기하고 재발급받은 신규 이용토큰(운영 KEY)를 사용하여야 합니다.
    1. 1. 사업자번호가 변경된 경우
    2. 2. 외부 노출이나 오용 등의 이유로 기존에 발급받은 운영 KEY를 폐기할 필요가 있는 경우

제7조(디지털서비스개방 이용시간 및 운영원칙)

  1. ① 운영기관은 365일 24시간 연중무휴로 플랫폼을 운영하는 것을 원칙으로 합니다. 다만, 다음 각 호의 경우 플랫폼의 전부 혹은 일부에 대한 운영이 중단 또는 제한될 수 있으며 그 사유가 해소되는 즉시 운영을 재개합니다.
    1. 1. 플랫폼 시스템의 정기점검 또는 증설 및 교체, 새로운 API 기능 적용 등을 위하여 필요한 경우
    2. 2. 국가비상사태, 정전 및 그에 준하는 상황이 발생한 경우
    3. 3. API 서비스 이용의 폭주 등으로 시스템에 장애가 발생한 경우
    4. 4. 그 밖에 플랫폼 운영 업무상 또는 기술상 사유로 플랫폼을 운영하기 어려운 상황이 발생한 경우
  2. ② 운영기관은 제1항 제1호에 따라 서비스 제공을 중단하거나 이용을 제한하는 경우에는 중단 및 제한 1주일 전, 제1항 제2호 내지 제4호에 따라 서비스 제공을 중단하거나 이용을 제한하는 경우에는 그 즉시 수요기관에 대해 중단 혹은 제한의 사유 및 기간 등에 관한 내용을 통지하여야 합니다.
  3. ③ 수요기관은 수요기관의 의무 위반 등 사정이 없는 이상 플랫폼에서 제공하는 API를 이용하여 플랫폼의 기능을 계속 이용할 수 있습니다.

제8x조(플랫폼 이용제한 및 해지)

  1. ① 운영기관은 규정 제20조에 따라 수요기관의 디지털서비스 개방 중단이 결정된 경우, 해당 수요기관에게 그 제한에 관한 내용을 제한시작 30일 전에 통지한 후 그 수요기관의 플랫폼 이용을 중단 또는 제한할 수 있습니다.
  2. ② 운영기관은 제5조제2항에 따른 등록 후 6개월 이상 이용자에게 서비스 제공을 지원한 기록이 없는 수요기관에 대하여 플랫폼과의 연계를 제한할 수 있습니다.
  3. ③ 수요기관은 제1항과 제2항에 따른 제한 조치에 대한 이의신청을 할 수 있으며, 운영기관이 이의신청이 정당하다고 인정하는 경우 수요기관은 플랫폼 이용을 재개할 수 있습니다.
  4. ④ 수요기관은 API 서비스 이용을 중단하고자 하거나 개방기관과의 협약이 종료된 경우에 규정의 별지 제3호 서식에 따른 디지털서비스 이용 변경 신청서를 작성하여 운영기관에게 이를 통지하여 플랫폼 이용을 해지할 수 있습니다.

제10조(서비스 장애 및 일시중지)

  1. ① 개방기관과 수요기관은 디지털서비스의 장애나 지연현상이 발생한 경우 즉시 운영기관에 알려야 합니다.
  2. ➁ 개방기관과 수요기관은 시스템의 정기점검 또는 증설 및 교체, API 변경 적용 등으로 디지털서비스 제공업무의 전부 또는 일부를 일시적으로 중지하고자 하는 경우 규정의 별지 제3호 서식에 따른 디지털서비스 이용 변경 신청서를 작성하여 최소 1주일 이전에 운영기관에 알려야 합니다.
  3. ➂ 제2항에 따라 일시중지를 결정한 수요기관의 장은 일시중지 시작일의 1주일 전까지 자신의 웹 또는 앱 등을 통하여 이용자에게 알려야 합니다.

제3장 운영기관 및 수요기관의 의무 등

제11조(운영기관의 의무)

  1. ① 운영기관은 플랫폼을 통한 API 서비스를 지속적이고 안정적으로 제공하기 위하여 노력하여야 합니다.
  2. ➁ 운영기관은 API 서비스 관련 불만 사항이 접수되는 경우 이를 신속하게 처리하여야 하며 신속한 처리가 곤란한 경우 그 사유와 처리 일정을 공지하거나 전자우편 등을 통하여 안내합니다.
  3. ③ 운영기관은 플랫폼과 수요기관의 API 연계 서버가 정상 상태인지 모니터링하고 관련 정보를 제공하여야 하며, 정상 상태가 아닌 경우 신속하게 복구하여야 합니다.
  4. ④ 운영기관은 개방기관으로부터 서비스 중단 사유 발생, 서비스 제공시 공지한 제반 사항, API 서비스 이용에 관한 변경사항 등을 통지 또는 통보받는 경우, 이를 수요기관에 사전 통보해야 합니다.

제12조(수요기관의 의무)

  1. ① 수요기관은 플랫폼을 이용함에 있어서 이 약관에서 규정하는 사항과 규정을 준수하여야 합니다.
  2. ② 수요기관은 API 서비스를 이용자에게 제공함에 있어서 다음과 같은 의무를 부담합니다.
    1. 1. API 서비스 업무담당자를 지정하고 이를 운영기관에게 통지하여야 합니다.
    2. 2. 발급받은 API Key를 선량한 관리자의 주의의무로 관리하고 타인에게 제공, 양도하거나 공개, 공유하여서는 아니됩니다.
    3. 3. 발급받은 API Key가 타인에 의하여 도용당하지 않도록 관리하여야 합니다.
    4. 4. 타인이 발급받은 API Key를 이용하여 플랫폼에 접근해서는 안 됩니다.
    5. 5. 개방기관과 체결한 별도의 협약이 없는 이상 API 서비스를 이용자에게 제공함에 있어 이용자로부터 별도의 금전적 대가나 정보 등 일체의 유무형의 대가를 받아서는 아니 됩니다.
    6. 6. 정보주체의 동의나 개방기관과 체결한 별도의 협약이 없는 이상 API 서비스 제공과 관련하여 취득한 개인(신용)정보의 임의 저장, 제3자 제공 및 재가공하여서는 아니됩니다.
    7. 7. 운영기관 또는 제공기관이 요청하는 경우 API 서비스의 이용 통계 정보를 제공하여야 합니다.
  3. ③ 수요기관은 플랫폼 이용신청 시 등록된 수요기관 및 업무담당자의 현행화된 정보를 유지하여야 합니다. 특히 운영기관의 통지를 수신할 수 있도록 전자우편주소 및 휴대전화번호 등 포털에 등록된 업무담당자의 연락처 정보를 항상 최신으로 유지하여야 합니다.

제13조(통지의 방법)

  1. ① 운영기관은 수요기관에 대한 통지가 필요한 경우 수요기관이 플랫폼 이용신청 시 등록된 업무담당자의 전자우편 혹은 휴대전화를 이용하여 통지할 수 있습니다.
  2. ② 불특정 다수 이용자에 대한 통지의 경우 포털에 게시함으로써 개별 통지를 갈음할 수 있습니다.

제14조(개인정보 보호 등)

  1. ① 운영기관이나 수요기관이 플랫폼의 이용과 관련하여 이용자의 개인정보를 처리하는 경우 「개인정보 보호법」, 「정보통신망 이용촉진 및 정보보호 등에 관한 법률」 등 관련 법령에 따라 처리합니다.
  2. ② 수요기관이 플랫폼 이용신청 시 운영기관에 제출한 기본정보 등 수요기관에 관한 정보는 수요기관이 API 서비스 제공 지원을 중단하거나 해지할 때까지 플랫폼에 보관됩니다.

제4장 사고 및 장애 대응

제15조(사고 및 장애에 대한 대응계획 수립 등)

  1. ① 수요기관은 다음 각 호의 사고에 대비한 대응계획 및 처리방안을 수립하여야 합니다.
    1. 1. 플랫폼 또는 포털과 연계된 전산기기 또는 통신회선 등의 장애 또는 가동중단으로 인하여 이용자에 대한 API 서비스가 중단되는 경우
    2. 2. API 서비스 제공과 관련된 전산보안시스템이 작동불능의 경우
    3. 3. API 서비스 제공과 연관된 정보가 누출되거나 민간서비스 시스템에 대한 해킹이 발생한 경우
    4. 4. 그 밖의 사유로 API 서비스 제공 업무에 장애가 발생한 경우
  2. ② 수요기관은 정상적인 API 서비스 제공에 문제가 발생한 경우 그에 대한 내용을 즉시 운영기관에게 통보하여야 합니다.

제5장 저작권 등

제16조(저작권의 귀속 등)

  1. ① 수요기관에게 제공하는 API 등에 대한 저작권은 행정안전부 또는 개방기관에 있습니다.
  2. ② 행정안전부는 해당 정책 및 플랫폼 운영상 필요한 경우 개방기관과 협의하여 API 서비스 등을 변경 할 수 있습니다.

제17조(책임의 제한)

  1. ① 행정안전부장관은 플랫폼 이용에 관하여 본 약관, 제반 규정, 이용 안내 및 관계 법령을 준수하지 않은 이용으로 인한 결과에 대하여 책임을 지지 않습니다.
  2. ② 행정안전부장관은 제7조제2항에 따른 플랫폼 운영 중단이나 제한으로 인하여 수요기관이 API 서비스를 이용하지 못하게 되거나 메시지나 통신 데이터가 손실되거나 전송되지 못하여 이용자나 수요기관에게 손해가 발생한 경우라도 행정안전부의 고의나 과실이 없는 한 그에 대한 책임을 부담하지 아니합니다.
  3. ③ 행정안전부장관은 API 서비스 제공과 관련하여 개방기관 또는 운영기관의 책임없는 사유로 수요기관과 이용자 사이에 발생한 분쟁에 대하여 개입할 의무가 없으며 이로 인한 손해를 배상할 책임이 없습니다.

제18조(약관 외 준칙)

  1. ① 행정안전부장관은 필요한 경우 개방기관 및 수요기관과 협의하여 특정 API 서비스에 관하여 적용될 사항을 정한 별도의 플랫폼 이용정책을 운영할 수 있습니다.
  2. ② 이 약관에서 정하지 아니한 사항은 행정안전부장관과 수요기관의 장이 협의하여 정한 바에 따릅니다.

제19조(준거법 및 재판관할)

  1. ① 플랫폼의 이용에 관하여 발생한 분쟁에 관한 소송은 대한민국 법령을 준거법으로 합니다.
  2. ② 플랫폼의 이용에 관하여 발생한 분쟁에 관한 소송은 대한민국 서울중앙지방법원을 전속관할로 합니다.
