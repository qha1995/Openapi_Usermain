표준 API

API 개요

표준 API는 표준 API 규격을 제공하여 수요기관과 개방기관이 이를 준용함으로써 디지털정보 서비스의 보안성을 기본적으로 확보하고 편이성을 극대화할 수 있도록 합니다.

전송 채널

수요기관 당 서비스 별 채널을 부여하며, 수요기관의 요청 IP와 API 키 쌍으로 전송채널을 관리

URI 계층구조

플랫폼에서 제공하는 API를 정의하고 식별 할 수 있도록 하는 URI 계층 구조는 다음과 같이 구성

URL 계층 구조
구분 Application

URL 계층 구조

<code><base path> / <institution code> / <service> / <resource></code>

<base path>: 플랫폼의 도메인 명을 기본으로 함

<institution code>: 기관코드로 플랫폼에서 관리되는 API 개방기관의 코드

<service>: 개방기관이 제공하는 API 서비스 명

<resource>: 정보제공자가 보유하고 있는 특정 자원을 요청하기 위한 식별 정보로써, 정보제공자가 제공해야 할 API를 특정

접속 보안 관리

서비스가 허가된 수요기관에게 접근 API 키를 발급하여 해당 수요기관만 서비스를 이용할 수 있도록 함

* API 키는 표준 암호화 모듈을 이용하여 암호화된 상태로 헤더에 담아서 전송

위변조 방지 방안

요청 헤더에 Body Data를 KISA SHA-256으로 해쉬 퍼리하여 해쉬값을 헤더에 등록하여 위변조 여부 점검

3. 헤더 구성 방안

수요기관의 허가 서비스 확인, 데이터의 위변조 여부 확인 등 요청의 검증을 위한 정보입니다.

3. 헤더 구성 방안
헤더 설명

Api-Key

RFC4122에서 명시한 버전 4 UUID를 사용한 범용 고유 식별자로 수요기관의 인증을 위한 헤더 값

* UUID,는 범용고유식별자로 32자리의 랜덤한 고유식별자를 사용, xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx 포맷을 사용하며 필요 시 즉시 생성

수요기관이 개방기관의 서비스를 이용하기 위해 포털을 통해 신청하여 발급받은 API 키 정보로서 플랫폼에 호출 시 헤더 정보에 해당 키를 기재하여 수요기관, 서비스를 식별할 수 있도록 함
Checksum

요청데이터를 KISA SHA-256 해쉬값으로 변환한 값으로 수요기관 요청 데이터를 위변조 검사하기 위한 헤더 값

* CHECKSUM은 송신측에서 전송할 모든 데이터를 16 비트 워드 단위로 구분하고, 1의 보수를 취하여 그 합에 대한 결과와 수신 측에서 같은 합에 대한 결과를 비교하여 일치 여부를 판단하여 데이터 위변조 여부를 확인함

* Body가 없는 요청 건은 공백처리

수요기관은 Body 전체 정보에 대한 KISA SHA-256 해쉬값(256BIT)을 구하여 체크섬으로 헤더에 등록하고 플랫폼에서는 해당 체크섬을 활용하여 데이터의 위변조 여부를 확인함
Enc-Key-Version

ARIA256 암호화 알고리즘을 사용한 암호화 키의 버전 정보 비교를 위한 헤더 값

* ARIA256은 경량 환경 및 하드웨어 구현을 위해 최적화된, Involutional SPN 구조를 갖는 범용 블록 암호 알고리즘으로 ARIA가 사용하는 대부분의 연산은 AOR과 같은 단순한 바이트 단위 연산으로 구성되어 있음

수요기관은 개방플랫폼에서 관리하는 암호화 키로 민감한 정보에 대해서 암호화를 하여 요청을 보내는데 해당 암호화 키의 버전 정보가 개방플랫폼에서 관리하는 버전과 일치하는지 확인함
Ins-Code

개방플랫폼에서 정의 관리하는 기관코드로 서비스의 요청 기관을 식별하기 위한 헤더 값

수요기관은 개방플랫폼에서 관리하는 수요기관 코드를 헤더에 등록하여 개방플랫폼에서 요청기관을 식별할 수 있도록 함

API 설명

API 키 & 암호화 키 발급

포털에서 발급받은 임시 API 키를 통해 운영 API 키와 암호화 키를 발급받을 수 있습니다. 이를 위한 API입니다.

API 키 재발급

서비스 별로 프로그램의 변경 및 배포, API 키의 노출 등의 일이 있을때, 이용기관은 플랫폼에서 API 키를 재발급(갱신)해 현행화해야합니다. 이를 위해 API 키를 재발급할 수 있도록 해주는 API입니다.

암호화키 갱신

이용기관, 제공기관은 개방플랫폼의 암호화 키가 갱신되었을 때 플랫폼의 API를 호출하여 암호화 키를 갱신해야합니다. 이를 위해 암호화 키를 요청할 수 있는 암호화 키 요청 API입니다.

수요기관 Health-Check

이용기관의 서비스 상태를 확인하기 위해 이용기관은 주기적으로 개방플랫폼의 Health-Check API를 호출하여 서비스가 정상 상태임을 통보해줘야 합니다. 이를 위해 이용기관이 요청하는 API입니다.

결제 결과 정보 제공

특정 기간에 이용기관에서 일어난 결제 정보를 제공기관에서 호출하여 결제 결과 정보를 수집하여 받을 수 있으며, 제공기관에서 원하는 데이터로 지정이 가능합니다.

이용약관 제공

제공기관에서 서비스 이용에 필요한 이용약관, 공지사항 등의 안내 정보를 제공할 수 있도록 연계하는 API입니다.
